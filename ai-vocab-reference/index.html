A note on CoPilot

"CoPilot" is a family. This sheet means M365 CoPilot — the chat surface inside Word, Excel, Outlook, and Teams, the one most execs encounter at work — unless flagged. Copilot Studio is the separate agent-building product, called out where relevant.

The biggest asymmetry in the sheet

M365 CoPilot's killer feature is auto-grounding in your company data. Claude and ChatGPT require explicit uploads. Gemini sits in between. For CoPilot users, knowledge-library work happens in M365 itself — folder structure, naming, permissions. For Claude/ChatGPT users, it's about uploading and structuring files into Projects.

Vocabulary alert

This is the row most likely to cause confusion. Teach "build a skill" and a CoPilot exec will hunt for a button labeled "skill" and find nothing. Translation: in CoPilot the same idea is an Agent (heavier, built in Copilot Studio) or a saved Prompt (lighter). Most execs only need the lighter version to start.

Tool → Collaborator → Delegate. Durable. Every tool maps to these.

How To Use This Sheet

Recognition, not recall

  • Distribute as a pre-read before Module 1. Identify which column is yours.
  • Reference during drills. When the lesson says "encode a workflow," point to the relevant row.
  • Expect a half-life. This sheet gets updated quarterly. The buttons move; the principles don't.
  • Don't memorize. The point is recognition, not recall. You look it up.
